The Yandex.Maps API consists of a large number of interconnected modules.
By default, when you enable the API, a standard set of modules is loaded (package.full), which includes everything that is necessary for the API.
To reduce the amount of API code that is loaded, you can connect certain modules instead of the standard set.
This can be done by specifying the necessary modules in the "load" GET parameter. The modules are separated by commas.
If the "load" GET parameter is not specified, package.full is loaded.
For more information about modules, see Modules.

In this example, to load only the modules, cards, placemarks, and a balloon geoobject.
